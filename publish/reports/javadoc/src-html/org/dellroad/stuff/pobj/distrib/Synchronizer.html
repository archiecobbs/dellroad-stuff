020/**
021 * Synchronizes a {@link PersistentObject} database across multiple nodes using {@code git(1)}.
022 *
023 * <p>
024 * This class provides a simple and lightweight way to build a "master-master" multi-node database
025 * based on {@link PersistentObject} object graphs. The basic idea is to apply {@code git(1)}'s
026 * efficient communication protocol and powerful merging algorithms to the {@link PersistentObject}
027 * database's XML representation. As a result, all nodes can make simultaneous changes, updates are
028 * communicated quickly, and in only a small corner case -- two conflicting changes to the same
029 * data element applied at virtually same time -- must one node's change "win" and the other's "lose".
030 * No global communication or two-phase commit protocol is required, and recovery from arbitrary
031 * network problems is automatic.
032 *
033 * <p>
034 * This class works by registering as a {@linkplain PersistentObjectListener listener} on a {@link PersistentObject}
035 * and also managing a private {@code git(1)} repository which is configured to track zero or more remotes.
036 * Each time a change occurs to the {@link PersistentObject} database, an updated XML copy of the database
037 * is committed to the {@code git(1)} repository, and then <i>synchronization</i> is performed: all remotes are fetched
038 * and their corresponding {@code git(1)} branches merged into the local {@code git(1)} branch (this process
039 * is performed by {@link #synchronize}). During synchronization, several merge strategies are tried until an XML file
040 * is produced that {@linkplain org.dellroad.stuff.pobj.PersistentObjectDelegate#validate validates}.
041 * If synchronization results in a non-trivial merge (i.e., any changes), the local {@link PersistentObject} is updated
042 * with the result and therefore made aware of the other nodes' changes. In this way, all nodes have their
043 * {@link PersistentObject} databases automatically synchronized and merged as they evolve.
044 *
045 * <p>
046 * The user of this class must separately initialize each node's {@code git(1)} repository, configure the
047 * {@linkplain #setRemotes remotes}, and determine when to {@link #synchronize} remote changes:
048 * as mentioned above, synchronization is performed automatically when the local {@link PersistentObject} database changes,
049 * but for remote changes either some external notification process must trigger a call to {@link #synchronize},
050 * or {@link #synchronize} must be invoked at regular intervals, or (for robustness in the face of transient
051 * network issues) both.
052 *
053 * <p>
054 * Merges are handled by {@link #merge merge()} (which subclasses may override if desired). The {@link #merge merge}
055 * implementation in this class attempts merges using the following strategies in order until there are no merge
056 * conflicts and a {@linkplain org.dellroad.stuff.pobj.PersistentObjectDelegate#validate valid} XML file is achieved:
057 *  <ul>
058 *  <li>Recursive merge using the patience algorithm</li>
059 *  <li>Same as previous, but resolving all conflicts one way or the other</li>
060 *  <li>Simply choosing one file or the other</li>
061 *  </ul>
062 *
063 * <p>
064 * If conflicts occur in attempt #1, but attempt #2 or #3 succeeds and the other node is the "winner", then
065 * {@link #handleConflictOverride handleConflictOverride()} is invoked to notify the local node that some of
066 * its recent changes have been overridden; if all three attempts fail, then
067 * {@link #handleImpossibleMerge handleImpossibleMerge()} is invoked and we stay with whatever version we are on;
068 * this latter case can only occur when another node is running a different version of the software
069 * (with different validation criteria).
070 *
071 * <p>
072 * For attempts #2 and #3, the "winner" is chosen by {@link #chooseConflictWinner chooseConflictWinner()} as follows:
073 *  <ul>
074 *  <li>If the commits have different timestamps, then the later commit wins</li>
075 *  <li>If the two commits were simultaneous, then the commit with the lexicographically higher SHA-1 checksum wins</li>
076 *  <li>If the two commits have the same timestamp and SHA-1 checksum, then there won't be any conflicts :)</li>
077 *  </ul>
078 *
079 * <p>
080 * Subclasses may {@linkplain #chooseConflictWinner override how the winner is chosen}.
081 *
082 * <p>
083 * Because all nodes follow this same algorithm for merging, they will all end up with the same result
084 * (note however that the {@link PersistentObject} object graph must serialize in a deterministic way for this to be true).
085 *
086 * <p>
087 * Note that merge conflicts can only happen when two nodes make changes at virtually the same time (or, when
088 * separated by a network partition): i.e., a change on one node occurs within the window of time after a change
089 * is made on another node but before the first node is notified and synchronizes. Using a simple
090 * real-time notification process, this window can be kept small, especially relative to the frequent-read,
091 * infrequent-write access pattern that the {@link PersistentObject} class is designed for. When a conflict override
092 * does occur, subclasses can override {@link #handleConflictOverride handleConflictOverride()} to take appropriate action.
093 *
094 * <p>
095 * Instances of this class are thread safe.
096 *
097 * @param <T> type of the root object
098 */
099public class Synchronizer<T> extends AbstractBean implements PersistentObjectListener<T> {

213    @Override
214    public void afterPropertiesSet() throws Exception {
215        super.afterPropertiesSet();
216
217        // Sanity check
218        if (this.persistentObject == null)
219            throw new Exception("no PersistentObject configured");
220        if (this.git == null)
221            throw new Exception("no GitRepository configured");
222        if (this.filename == null)
223            throw new Exception("no filename configured");
224        if (this.branch == null)
225            throw new Exception("no branch name configured");
226        if (!GitRepository.BRANCH_NAME_PATTERN.matcher(this.branch).matches())
227            throw new Exception("illegal branch name `" + this.branch + "'");
228        if (this.remotes == null)
229            throw new Exception("no remotes configured");
230        for (String remote : this.remotes) {
231            if (remote == null || !GitRepository.REMOTE_NAME_PATTERN.matcher(remote).matches())
232                throw new Exception("illegal remote name `" + remote + "'");
233        }
234
235        // Listen for POBJ changes
236        this.persistentObject.addListener(this);
237
238        // Do the first update
239        final PersistentObject<T>.Snapshot snapshot = this.persistentObject.getSharedRootSnapshot();
240        this.applyLocalUpdate(snapshot, "Initial update after startup (version " + snapshot.getVersion() + ")");
241    }

262    /**
263     * Given a (possibly) updated local root, {@linkplain #commit commit} it to our local branch,
264     * {@link #synchronize} with our peers if it represented a change, and apply the newly merged
265     * root (if any) back to the associated {@link PersistentObject}.
266     *
267     * <p>
268     * This method is invoked at {@linkplain #afterPropertiesSet startup} and after {@linkplain #handleEvent notification}
269     * of each local change.
270     *
271     * <p>
272     * This method does nothing if the root is null.
273     *
274     * @param snapshot snapshot containing new root (possibly null) and its version
275     * @param commitMessage message for the commit
276     */
277    protected synchronized void applyLocalUpdate(PersistentObject<T>.Snapshot snapshot, String commitMessage) {
278
279        // Get root; ignore if null
280        final T root = snapshot.getRoot();
281        if (root == null)
282            return;
283
284        // Commit new root
285        this.log.debug("committing new local root to branch `" + this.branch + "' (version " + snapshot.getVersion() + ")");
286        String commit;
287        try {
288            commit = this.commit(root, commitMessage);
289        } catch (Exception e) {
290            this.log.error("error committing local root to branch `" + this.branch + "'", e);
291            return;
292        }
293
294        // If nothing changed, nothing else to do
295        if (commit == null) {
296            this.log.debug("commit of version " + snapshot.getVersion() + " resulted in no change on branch `" + this.branch + "'");
297            return;
298        }
299        this.log.info("committed root version " + snapshot.getVersion() + " as " + commit + " on branch `" + this.branch + "'");
300
301        // Synchronize it; if no merge required or merge was trivial, we're done
302        if (!this.synchronize())
303            return;
304
305        // Read in the new, merged object graph
306        final ArrayList<T> rootList = new ArrayList<T>(1);
307        String mergeCommit;
308        try {
309            mergeCommit = this.git.access(this.branch, new GitRepository.Accessor() {
310                @Override
311                public void accessWorkingCopy(File dir) {
312                    rootList.add(Synchronizer.this.readXMLFile(dir, false));
313                }
314            });
315        } catch (PersistentObjectException e) {
316            this.log.error("error reading newly merged root; the local persistent object will not be updated", e);
317            return;
318        }
319
320        // Apply changes to local persistent object; but if the local version has changed, do nothing;
321        // we will do another merge when we get the next notification for the newer root.
322        try {
323            this.log.debug("applying merge commit " + mergeCommit + " to local persistent object");
324            final long version = this.persistentObject.setRoot(rootList.get(0), snapshot.getVersion());
325            this.log.info("successfully applied commit " + mergeCommit + " to local persistent object as version " + version);
326        } catch (PersistentObjectVersionException e) {
327            this.log.debug("merged root is out of date (version " + e.getExpectedVersion() + " < " + e.getActualVersion()
328              + "); will wait for next notification");
329        } catch (PersistentObjectException e) {
330            this.log.error("error applying newly merged root; the local persistent object will not be updated", e);
331        }
332    }

353    /**
354     * Commit the given root to our {@code git(1)} repository.
355     *
356     * <p>
357     * If the local branch does not already exist, it will be created.
358     *
359     * @param root persistent object root
360     * @param commitMessage message for the commit
361     * @return new commit ID if commit occurred, or null if {@code root} was identical to the already-committed head of our branch
362     * @throws IllegalArgumentException if {@code root} is null
363     * @throws IllegalArgumentException if {@code commitMessage} is null
364     */
365    protected synchronized String commit(final T root, String commitMessage) {
366
367        // Sanity check
368        if (root == null)
369            throw new IllegalArgumentException("null root");
370        if (commitMessage == null)
371            throw new IllegalArgumentException("null commitMessage");
372
373        // Ensure branch exists
374        this.git.ensureBranch(this.branch, "Empty commit as the basis for branch `" + this.branch + "'");
375
376        // Get previous commit ID
377        final String previousCommit = this.git.followReference("refs/heads/" + this.branch);
378
379        // Commit it
380        this.log.debug("committing root to local branch `" + this.branch + "'");
381        final String newCommit = this.git.commit(this.branch, new GitRepository.Accessor() {
382            @Override
383            public void accessWorkingCopy(File dir) {
384                PersistentObject.write(root, Synchronizer.this.persistentObject.getDelegate(),
385                  Synchronizer.this.getXMLFile(dir));
386            }
387        }, commitMessage);
388
389        // Report results
390        return !newCommit.equals(previousCommit) ? newCommit : null;
391    }

393    /**
394     * Synchronize what we have previously committed with our peers' latest information.
395     *
396     * <p>
397     * This method fetches the configured remotes and then merges them into our local branch,
398     * ensuring that the merge always yeilds a root that validates. If the fetch fails for
399     * one or more remotes, an error is logged and synchronization proceeds anyway.
400     *
401     * <p>
402     * If the local branch does not already exist, it will be created.
403     *
404     * <p>
405     * This method does not affect the {@link PersistentObject} associated with this instance;
406     * the caller is responsible for doing that if necessary (i.e., when true is returned).
407     *
408     * <p>
409     * This method delegates to {@link #merge merge} to do the actual merging.
410     *
411     * @return true if one or more non-trivial merges ocurred, false if we were already up-to-date
412     */
413    public synchronized boolean synchronize() {
414
415        // Fetch the latest from our remotes
416        this.log.debug("beginning synchronization with remotes: " + this.remotes);
417        try {
418            this.git.fetch(this.remotes);
419        } catch (GitException e) {
420            this.log.info("pre-synchronize git fetch failed on one or more remotes; proceeding anyway");
421        }
422
423        // Ensure local branch exists
424        this.git.ensureBranch(this.branch, "Empty commit as the basis for branch `" + this.branch + "'");
425
426        // Merge our branch with each remote's branch
427        int numMerges = 0;
428        int numFails = 0;
429    remoteLoop:
430        for (String remote : this.remotes) {
431
432            // Get remote commit ID and date
433            final String remoteRef = "refs/remotes/" + remote + "/" + this.branch;
434            if (!this.git.getRepoFile(remoteRef).exists()) {
435                this.log.debug("remote `" + remote + "' branch `" + this.branch + "' does not exist yet, skipping this remote");
436                continue;
437            }
438            final String remoteCommit = this.git.followReference(remoteRef);
439
440            // Get local commit ID and date
441            final String localRef = "refs/heads/" + this.branch;
442            final String localCommit = this.git.followReference(localRef);
443
444            // Compare trees for equality
445            if (this.git.equalTrees(localCommit, remoteCommit)) {
446                this.log.debug("remote `" + remote + "' commit " + remoteCommit + " matches our branch `"
447                  + this.branch + "' commit " + localCommit + ", skipping this remote");
448                continue;
449            }
450
451            // Attempt merge
452            this.log.info("merging remote `" + remote + "' commit " + remoteCommit + " with local commit " + localCommit);
453            final String commit = this.merge(remote, localCommit, remoteCommit, this.branch);
454            if (commit == null) {
455                numFails++;
456                continue;
457            }
458
459            // Was a non-trivial merge required?
460            if (!commit.equals(localCommit))
461                numMerges++;
462        }
463
464        // Done
465        this.log.debug("completed synchronization with remotes " + this.remotes
466          + " resulting in " + numMerges + " non-trivial merge(s) and " + numFails + " merge failure(s)");
467        return numMerges > 0;
468    }

470    /**
471     * Attempt to merge a local and remote commit and commit the result to the specified branch.
472     *
473     * <p>
474     * The implementation in {@link Synchronizer} attempts to merge the files in an automated way
475     * using git's patience merge algorithm.
476     *
477     * <p>
478     * In case of conflicts, {@link #chooseConflictWinner chooseConflictWinner()},
479     * {@link #handleConflictOverride handleConflictOverride()} and/or
480     * {@link #handleImpossibleMerge handleImpossibleMerge()} is invoked as appropriate.
481     *
482     * @param remote name of the remote being merged
483     * @param localCommit local commit name (SHA-1 hash)
484     * @param remoteCommit remote commit name (SHA-1 hash)
485     * @param commitBranch branch to commit merged result to
486     * @return commit name (SHA-1 hash) if the merge succeeded, null if the merge failed
487     */
488    protected String merge(String remote, String localCommit, String remoteCommit, String commitBranch) {
489
490        // Determine who is the winner in case of conflicts
491        boolean iWin = localCommit != null && this.chooseConflictWinner(localCommit, remoteCommit);
492
493        // Build merge strategy list
494        final MergeStrategy[] strategyList = new MergeStrategy[] {
495          MergeStrategy.RECURSIVE_PATIENCE,
496          iWin ? MergeStrategy.RECURSIVE_PATIENCE_OURS : MergeStrategy.RECURSIVE_PATIENCE_THEIRS,
497          iWin ? MergeStrategy.RECURSIVE_OURS : MergeStrategy.RECURSIVE_THEIRS,
498          iWin ? MergeStrategy.OURS : MergeStrategy.THEIRS
499        };
500
501        // Attempt merges using successively less "mergey" strategies
502        for (MergeStrategy strategy : strategyList) {
503
504            // Attempt merge
505            this.log.debug("attempting merge of remote `" + remote + "' commit " + remoteCommit
506              + " with local commit " + localCommit + " using strategy " + strategy);
507            String commit;
508            try {
509                commit = this.git.merge(commitBranch, remoteCommit, strategy, new GitRepository.Accessor() {
510                    @Override
511                    public void accessWorkingCopy(File dir) {
512                        Synchronizer.this.log.debug("validating merged file `" + Synchronizer.this.getXMLFile(dir) + "'");
513                        Synchronizer.this.readXMLFile(dir, true);
514                    }
515                }, "Merged remote `" + remote + "' commit " + remoteCommit + " using strategy " + strategy);
516            } catch (GitMergeConflictException e) {
517                this.log.debug("merge using strategy " + strategy + " failed with conflict(s)");
518                continue;
519            } catch (PersistentObjectException e) {
520                this.log.debug("merge using strategy " + strategy + " did not validate: " + e);
521                continue;
522            }
523
524            // We have a winner
525            if (commit.equals(localCommit)) {
526                this.log.debug("merge of remote `" + remote + "' commit " + remoteCommit + " with local commit "
527                  + localCommit + " using strategy " + strategy + " resulted in no change to our version");
528            } else {
529                this.log.debug("successfully merged remote `" + remote + "' commit " + remoteCommit + " with local commit "
530                  + localCommit + " using strategy " + strategy + " resulting in commit " + commit);
531            }
532
533            // Log a warning if some of our local changes got overridden by remote
534            switch (strategy) {
535            case RECURSIVE_PATIENCE_THEIRS:
536            case RECURSIVE_THEIRS:
537                this.handleConflictOverride(remote, localCommit, remoteCommit, strategy);
538                break;
539            default:
540                break;
541            }
542
543            // Done
544            return commit;
545        }
546
547        // Nothing worked - so we must be running different versions of the application
548        this.handleImpossibleMerge(remote, localCommit, remoteCommit);
549        return null;
550    }

552    /**
553     * Choose which commit should "win" in case there is a conflict between two commits.
554     *
555     * <p>
556     * It is imperative that this method <b>only</b> depend on information associated with the given commits.
557     * In particular, it must not depend on which node it is running on. This ensures that all nodes come
558     * to the same conclusion and therefore produce the same outcome when merging.
559     *
560     * <p>
561     * The implementation in {@link Synchronizer} chooses the commit with the later timestamp, or if both commits
562     * have the same timestamp, then the commit with the lexicographically higher SHA-1 checksum.
563     *
564     * @param commit1 first commit ID
565     * @param commit2 second commit ID
566     * @return true if {@code commit1} should win, false if {@code commit2} should win
567     */
568    protected boolean chooseConflictWinner(String commit1, String commit2) {
569        final Date date1 = this.git.getAuthorDate(commit1);
570        final Date date2 = this.git.getAuthorDate(commit2);
571        return date1.compareTo(date2) > 0 || (date1.compareTo(date2) == 0 && commit1.compareTo(commit2) > 0);
572    }

576     * in the end we were able to produce a valid root.
577     *
578     * <p>
579     * The implementation in {@link Synchronizer} just logs a warning message.
580     *

582     * @param localCommit our commit in the merge
583     * @param remoteCommit remote's commit in the merge
584     * @param strategy the strategy that was used to resolve the conflict
585     */
586    protected void handleConflictOverride(String remote, String localCommit, String remoteCommit, MergeStrategy strategy) {
587        this.log.warn("some changes in local commit " + localCommit + " have been overridden by changes from remote `"
588          + remote + "' commit " + remoteCommit + " using merge strategy " + strategy);
589    }

593     * This can only happen when a remote node has different validation criteria than we do, for example,

596     * <p>

606    }

608    /**
609     * Get the XML file.
610     *
611     * <p>
612     * To avoid race conditions, the file should only be accessed from within a {@link GitRepository.Accessor} callback.
613     *
614     * @param dir {@code git(1)} working directory root
615     * @return persistent object XML file
616     */
617    protected File getXMLFile(File dir) {
618        return new File(dir, this.filename);
619    }

624     * <p>

628     * @param validate whether to also validate the file
629     * @return decoded root object
