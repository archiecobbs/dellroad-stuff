034/**
035 * Manages an associated Spring {@link WebApplicationContext} with each {@link VaadinSession} (aka, "Vaadin application").
036 * Typically created implicitly via {@link SpringVaadinServlet}.
037 *
038 * <h3>Overview</h3>
039 *
040 * <p>
041 * In Vaadin 7, the {@link com.vaadin.server.VaadinSession} object holds the state associated with each client browser
042 * connection to a Vaadin servlet. For consistency with older versions of Vaadin, we'll call this a "Vaadin application" instance.
043 * This class gives each Vaadin such "Vaadin application" instance its own Spring application context, and all such
044 * application contexts share the same parent context, which is the one associated with the overal servlet web context
045 * (i.e., the one created by Spring's {@link org.springframework.web.context.ContextLoaderListener ContextLoaderListener}).
046 * This setup is analogous to how Spring's {@link org.springframework.web.servlet.DispatcherServlet DispatcherServlet}
047 * creates per-servlet application contexts that are children of the overall servlet web context.
048 *
049 * <p>
050 * This class is implemented as a {@link SessionInitListener} and {@link SessionDestroyListener} on the servlet's
051 * {@link com.vaadin.server.VaadinService} object. In turn, the Spring context is created when a new Vaadin application
052 * instance is initialized, and destroyed when it is closed. To use this class, use the {@link SpringVaadinServlet}
053 * in place of the usual {@link com.vaadin.server.VaadinServlet} in {@code web.xml}.
054 *
055 * <h3>Accessing the Spring Context</h3>
056 *
057 * <p>
058 * The {@link #getApplicationContext() getApplicationContext()} method provides access to the application context.
059 * Alternately, use {@link VaadinConfigurable @VaadinConfigurable} (see below) and implement
060 * {@link org.springframework.context.ApplicationContextAware}, etc.
061 * Invoking {@link #configureBean configureBean()} at any time will configure a bean manually.
062 *
063 * <h3>Exposing the Vaadin Session</h3>
064 *
065 * <p>
066 * The {@link VaadinSession} instance representing the "Vaadin application" can be exposed in the associated Spring
067 * application context and therefore made available for autowiring, etc. Simply add a bean definition that invokes
068 * {@link VaadinUtil#getCurrentSession}:
069 * <blockquote><pre>
070 *  &lt;bean id="vaadinSession" class="org.dellroad.stuff.vaadin7.VaadinUtil" factory-method="getCurrentSession"/&gt;
071 * </pre></blockquote>
072 * This bean can then be autowired into application-specific "backend" beans, allowing them to use e.g.
073 * {@link VaadinUtil#invoke VaadinUtil.invoke()}, which performs the locking necessary to avoid race conditions.
074 * But see also {@link VaadinApplication} for a convenience class that makes this process a little cleaner.
075 *
076 * <h3><code>@VaadinConfigurable</code> Beans</h3>
077 *
078 * <p>
079 * It is also possible to configure beans outside of this application context using AOP, so that any invocation of
080 * {@code new FooBar()}, where the class {@code FooBar} is marked {@link VaadinConfigurable @VaadinConfigurable},
081 * will automagically cause the new {@code FooBar} object to be configured by the application context associated with
082 * the currently running Vaadin application. In effect, this does for Vaadin application beans what Spring's
083 * {@link org.springframework.beans.factory.annotation.Configurable @Configurable} does for regular servlet context-wide beans.
084 *
085 * <p>
086 * Note however that Spring {@linkplain org.springframework.beans.factory.DisposableBean#destroy destroy methods}
087 * will not be invoked on application close for these beans, since their lifecycle is controlled outside of the
088 * Spring application context (this is also the case with
089 * {@link org.springframework.beans.factory.annotation.Configurable @Configurable} beans). Instead, these beans
090 * can themselves register as a {@link SessionDestroyListener} for shutdown notification; but see
091 * {@link VaadinUtil#addSessionDestroyListener VaadinUtil.addSessionDestroyListener()} for a memory-leak free
092 * method for doing this.
093 *
094 * <h3>Serialization and Clustering</h3>
095 *
096 * <p>
097 * Instances are serializable; on deserialization the {@link ConfigurableWebApplicationContext} associated with the
098 * {@link VaadinSession} is {@linkplain ConfigurableWebApplicationContext#refresh refreshed}; therefore, the
099 * {@link ConfigurableWebApplicationContext} is not itself stored in the HTTP session by this class. This is consistent
100 * with the way normal Spring application contexts ususally work across clustered servers.
101 *
102 * <p>
103 * However, any session-scope beans should work as expected.
104 * So while this class associates an application context with each {@link VaadinSession}, when sessions are shared across
105 * multiple servers in a clustered environment, there will actually be a separate application contexts per server.
106 * Beans that must truly be "session wide" should be declared {@code scope="session"} as you normally would.
107 *
108 * <p>
109 * Note: using {@code scope="session"} requires adding a {@code <listener>} clause registering Spring's
110 * {@link org.springframework.web.context.request.RequestContextListener} in your {@code web.xml}.
111 *
112 * @see VaadinConfigurable
113 * @see SpringVaadinServlet
114 * @see VaadinApplication
115 * @see com.vaadin.server.VaadinService
116 * @see com.vaadin.server.VaadinSession
117 */
118public class SpringVaadinSessionListener implements SessionInitListener, SessionDestroyListener {
119
120    private static final long serialVersionUID = -2107311484324869198L;
121    private static final WeakHashMap<VaadinSession, ConfigurableWebApplicationContext> CONTEXT_MAP = new WeakHashMap<>();
122
123    private final Logger log = LoggerFactory.getLogger(this.getClass());
124    private final UUID uuid = UUID.randomUUID();            // ensures a unique ID for the associated context across a cluster
125    private final String applicationName;
126    private final String configLocation;
127
128// Constructor
129
130    /**
131     * Constructor.
132     *
133     * @param applicationName Vaadin application name
134     * @param configLocation XML application context config file location(s), or null for the default
135     *  {@code /WEB-INF/ServletName.xml}, where {@code ServletName} is the value of {@code applicationName}
136     * @throws IllegalArgumentException if {@code applicationName} is null
137     */
138    public SpringVaadinSessionListener(String applicationName, String configLocation) {
139        if (applicationName == null)
140            throw new IllegalArgumentException("null applicationName");
141        this.applicationName = applicationName;
142        this.configLocation = configLocation;
143    }
144
145// Public API
146
147    /**
148     * Get the name of this Vaadin application.
149     */
150    public String getApplicationName() {
151        return this.applicationName;
152    }
153
154    /**
155     * Get the Spring application context associated with the given {@link VaadinSession}.
156     *
157     * @return Spring application context, or null if none is found
158     * @throws IllegalArgumentException if {@code session} is null
159     */
160    public static ConfigurableWebApplicationContext getApplicationContext(VaadinSession session) {
161        if (session == null)
162            throw new IllegalArgumentException("null session");
163        return CONTEXT_MAP.get(session);
164    }
165
166    /**
167     * Get the Spring application context associated with the current thread's {@link VaadinSession}.
168     *
169     * @return Spring application context, never null
170     * @throws IllegalStateException if there is no {@link VaadinSession} associated with the current thread
171     * @throws IllegalStateException if there is no Spring application context associated with the {@link VaadinSession}
172     */
173    public static ConfigurableWebApplicationContext getApplicationContext() {
174
175        // Get current session
176        VaadinSession session = VaadinUtil.getCurrentSession();
177
178        // Get the associated application context
179        ConfigurableWebApplicationContext context = SpringVaadinSessionListener.getApplicationContext(session);
180        if (context == null) {
181            throw new IllegalStateException("there is no Spring application context associated with the current"
182              + " VaadinSession; are you using SpringVaadinServlet instead of VaadinServlet?");
183        }
184
185        // Done
186        return context;
187    }
188
189    /**
190     * Configure the given bean using the Spring application context associated with the current thread's {@link VaadinSession}.
191     *
192     * @param bean Java bean to configure
193     * @throws IllegalStateException if there is no {@link VaadinSession} associated with the current thread
194     * @throws IllegalStateException if there is no Spring application context associated with the {@link VaadinSession}
195     */
196    public static void configureBean(Object bean) {
197        final BeanConfigurerSupport beanConfigurerSupport = new BeanConfigurerSupport();
198        beanConfigurerSupport.setBeanFactory(SpringVaadinSessionListener.getApplicationContext().getBeanFactory());
199        beanConfigurerSupport.afterPropertiesSet();
200        beanConfigurerSupport.configureBean(bean);
201        beanConfigurerSupport.destroy();
202    }
203
204// SessionInitListener
205
206    @Override
207    public void sessionInit(final SessionInitEvent event) throws ServiceException {
208        VaadinUtil.invoke(event.getSession(), new Runnable() {
209            @Override
210            public void run() {
211                SpringVaadinSessionListener.this.loadContext(event.getSession(), event.getRequest());
212            }
213        });
214    }
215
216// Context loading
217
218    /**
219     * Load the Spring application context.
220     *
221     * <p>
222     * This method expects that {@code session} is already {@linkplain VaadinSession#getLockInstance locked}.
223     *
224     * @param session the Vaadin application session
225     * @param request the triggering request
226     */
227    protected void loadContext(VaadinSession session, VaadinRequest request) {
228
229        // Sanity check
230        if (session == null)
231            throw new IllegalStateException("null session");
232        VaadinUtil.assertSession(session);
233        if (request == null)
234            throw new IllegalStateException("null request");
235        if (SpringVaadinSessionListener.getApplicationContext(session) != null)
236            throw new IllegalStateException("context already loaded for session " + session);
237
238        // Logging
239        this.log.info("creating new application context for Vaadin application [" + this.getApplicationName()
240          + "] in session " + session);
241
242        // Find the servlet context parent application context
243        String contextPath = "/";
244        ServletContext servletContext = null;
245        WebApplicationContext parent = null;
246        if (request instanceof VaadinServletRequest) {
247            contextPath = ((VaadinServletRequest)request).getHttpServletRequest().getContextPath() + "/";
248            servletContext = ((WrappedHttpSession)session.getSession()).getHttpSession().getServletContext();
249            parent = WebApplicationContextUtils.getWebApplicationContext(servletContext);
250        } else if (request instanceof VaadinPortletRequest)
251            this.log.warn("portlets are not supported yet");
252        else
253            this.log.warn("unsupported VaadinRequest instance: " + request);
254
255        // Create and configure a new application context for this Application instance
256        final XmlWebApplicationContext context = new XmlWebApplicationContext();
257        context.setId(ConfigurableWebApplicationContext.APPLICATION_CONTEXT_ID_PREFIX
258          + contextPath + this.getApplicationName() + "-" + this.uuid);
259        if (parent != null)
260            context.setParent(parent);
261        if (servletContext != null)
262            context.setServletContext(servletContext);
263        //context.setServletConfig(??);
264        context.setNamespace(this.getApplicationName());
265
266        // Set explicit config location(s), if any
267        if (this.configLocation != null)
268            context.setConfigLocation(this.configLocation);
269
270        // Associate context with the current session
271        boolean success = false;
272        CONTEXT_MAP.put(session, context);
273
274        // Refresh context
275        try {
276            context.refresh();
277            success = true;
278        } finally {
279            if (!success)
280                CONTEXT_MAP.remove(session);
281        }
282    }
283
284// SessionDestroyListener
285
286    @Override
287    public void sessionDestroy(SessionDestroyEvent event) {
288        final VaadinSession session = event.getSession();
289        final ConfigurableWebApplicationContext context = SpringVaadinSessionListener.getApplicationContext(session);
290        if (context == null) {
291            this.log.info(this.getClass().getSimpleName() + ".sessionDestroy() invoked but no application context found"
292              + " for Vaadin application [" + SpringVaadinSessionListener.this.getApplicationName() + "]");
293            return;
294        }
295        CONTEXT_MAP.remove(session);
296        VaadinUtil.invoke(session, new Runnable() {
297            @Override
298            public void run() {
299                SpringVaadinSessionListener.this.log.info("closing Vaadin application ["
300                  + SpringVaadinSessionListener.this.getApplicationName() + "] application context: " + context);
301                context.close();
302            }
303        });
304    }
305
306// Serialization
307
308    private void readObject(ObjectInputStream input) throws IOException, ClassNotFoundException {
309        input.defaultReadObject();
310        final VaadinSession session = VaadinUtil.getCurrentSession();
311        final VaadinRequest request = VaadinUtil.getCurrentRequest();
312        VaadinUtil.invoke(session, new Runnable() {
313            @Override
314            public void run() {
315                SpringVaadinSessionListener.this.loadContext(session, request);
316            }
317        });
318    }
