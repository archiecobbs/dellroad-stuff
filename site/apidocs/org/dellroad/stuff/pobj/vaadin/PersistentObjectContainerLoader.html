Class PersistentObjectContainerLoader<T,K>

  • java.lang.Object
    • org.dellroad.stuff.pobj.vaadin.PersistentObjectContainerLoader<T,K>
  • Type Parameters:
    T - type of the persistent object
    K - the type of the Java objects that back each Item in the container

    public class PersistentObjectContainerLoader<T,K>
extends Object
    Facilitates populating a AbstractSimpleContainer with data from PersistentObject and automatically updating it when that PersistentObject changes.

    The constructor requires a PersistentObjectContainerLoader.Generator to generate, from the PersistentObject root object, the Java objects that populate the AbstractSimpleContainer.

    Note: Instances of this class start out in a non-listening state. You must invoke connect() to start listening to the PersistentObject and (re)load the container contents. You should invoke disconnect() whenever the container is no longer used to avoid a memory leak. Typically, connect() and disconnect() are invoked from the attach() and detach() methods (respectively) of the GUI widget that is using the container.

        public void connect(PersistentObject<T> persistentObject)
        Register this container as a listener on the persistent object and load the container. This method may only be invoked while the VaadinSession is locked by the current thread. If this method is invoked twice with no intervening disconnect(), this instance will automatically disconnect from the old PersistentObject before connecting to the new one.
        Parameters:
        persistentObject - persistent object database
        Throws:
        IllegalArgumentException - if persistentObject is null
        IllegalStateException - if there is no VaadinSession associated with the current thread
