      • handlePersistentObjectChange

        protected abstract void handlePersistentObjectChange(T oldRoot,
                                T newRoot,
                                long version)
        Handle an update of the PersistentObject root object. The current thread will already have the Vaadin session locked, so it should not invoke handleEvent() to handle the event.

        The caller should not modify oldRoot or newRoot.

        Parameters:
        oldRoot - previous root object; may be null if empty starts or stops are supported
        newRoot - new root object; may be null if empty starts or stops are supported
        version - version corresponding to newRoot
