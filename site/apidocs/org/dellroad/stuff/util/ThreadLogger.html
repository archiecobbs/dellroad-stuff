    public class ThreadLogger
extends WriterAppender
    A Log4J appender that can be configured on a per-thread basis to direct the log messages to an arbitrary Writer destination.

    Normally this appender does nothing. However, if a thread invokes setLogDestination() with a non-null Writer, then any messages logged by that particular thread will go to that Writer. This allows individual threads to copy and/or redirect their own log messages as they see fit.

    Configure this appender just as you would any other. For example: 

      <!-- Per-thread logger -->
  <appender name="threadlog" class="org.dellroad.stuff.util.ThreadLogger">
      <param name="Threshold" value="debug"/>
      <layout class="org.apache.log4j.PatternLayout">
          <param name="ConversionPattern" value="%d{ISO8601} %p: %m%n"/>
      </layout>
  </appender>

  ...

  <root>
      ...
      <appender-ref ref="threadlog">
  </root>
